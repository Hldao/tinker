# 工坊 / 账号页重塑

> 起草: 2026-06-08 · 状态: 待实施

## 一、为什么动

之前「我的工作室」表面是个人页 · 实际是 LinkedIn 风名片(@handle + tagline + 累计数字 + 工具栏) 加项目列表。问题三层:

1. **语义错位**。"工作室"既指个人 (workshop) 也指团队 (studio) · 一词两意。工坊主人自己看 @daodao 的工作室时还能看到「挂靠 → 捣鼓团队 工作室」一行 · 脑子转不过来。
2. **账号设置无家可归**。CLI 钥匙塞在工坊头部当 chip · 编辑 handle 走"编辑工作室"按钮 · 通知偏好 webapp 完全没暴露 (CLI 端有 muted / cooldown / dismissedToday 状态 · 用户在网页上摸不到)。
3. **工坊空间感缺失**。头部没有"这是个工坊"的叙事。进店一眼名片 + 数字 · 跟 GitHub profile 长得一样。

## 二、重塑方案

### §1 命名

个人页 workshop → 「工坊」 · 团队 studio 继续叫「工作室」。

- 导航按钮: "我的工作室" → "我的工坊"
- 工坊页头部标签: "你的工作室 / 一间工作室" → "你的工坊 / 一间工坊"
- mention 文案: "@daodao 的工作室" → "@daodao 的工坊"
- 团队 studio 相关全部文案不动

内部命名 (代码里的 `workshop` 变量 / route `#/w/` / API path) 全部保留 · 只换前台显示文案。

### §2 入口

主导航 5 项不增不减。新增账号页 · 入口两条:

1. 顶部 dateline 那一行的 @handle (现在是死字) 改成可点 · 点开进账号页
2. 工坊页头部右上一个"账号 →"小字链接

账号页 route 用 `#/account` · 不进主导航。

### §3 工坊页头部

重塑后形态:

```
─────────────────────────────────
你的工坊                  [账号 →]
@daodao
「一句话: 我在做什么」
工作台上在打磨 3 个 · 最近一次开张 2 天前
常用 · jsx · sqlite · claude
挂靠 → 捣鼓团队 工作室
看看复用累计 →
─────────────────────────────────
```

字段语义:

- **工作台上在打磨 N 个** = in-flight 项目数 (status=active 或 stuck · 不含 stale 沉到"也在做"的)。N=0 时显示"工作台还空着"
- **最近一次开张** = 最近一次 ship 时间 · shortAgo 格式。从未 ship 过显示"还没开过张"
- **常用** = 项目 tools 频次 top 3。没工具记录就不显示这一行
- **挂靠** = 现有逻辑保留 (列已挂靠的 studios + isMe 时加 "再建一个")
- **看看复用累计** = 默认折叠 chip · 点开横排展开:

```
跑通 12 · 在做 3 · 启发了 8 人动手 · 方法被用 14 · 跟进过 5    [收起]
            这是只有自己看到的
```

只在 `isMe` 时显示这个 chip · 访客看不到。

#### 从工坊头部清出去的

- 「编辑」按钮 (动 handle / tagline) → 搬到账号页 §5.1
- 「CLI 钥匙」按钮 → 搬到账号页 §5.3
- mini stats 那一长串排比 → 收进折叠 chip

### §4 工坊页主体

保持现有结构不动:

- 最近在动
- 也在做
- 做过的 (索引 chip 行)
- 沉淀方法
- 跟进的

这几个 section 的渲染逻辑 / 视觉 / 排序都不动。本次重塑只动头部。

### §5 账号页

route `#/account` · 新建一个 view。旧报刊单栏布局 · 不做 SaaS 风的左侧 tab + 右侧 panel。从上到下六栏:

#### 5.1 基本资料

handle · tagline · 邮箱 (read-only · 改邮箱走 magic link 流程) 。

行内编辑 · 不再走弹窗。改 handle 仍然走 renameHandle API · 错误提示就地显示。

#### 5.2 挂靠的工作室

列出当前用户挂靠的 studios (跟工坊头部那一行同源数据)。每行一个 studio · 显示 name + 我的角色 (owner / member)。底部"+ 建一个" 和 "接受邀请"两个按钮。

工坊头部的"挂靠 → ..."一行**同时保留** · 那是对外身份信号 · 这里是管理入口。

#### 5.3 CLI 钥匙

把现有 `modal-api-tokens` 弹窗的内容原样搬过来当一栏。"生成新钥匙 / 现有钥匙列表 / 撤销 / 复制 AI 接入指南" 这些都不动。弹窗本身保留但不再从工坊页打开 · 仅作为 fallback。

#### 5.4 通知偏好

本次 P0 仅做 UI 空壳 · 显示当前状态 (从 CLI 端读 `~/.tinker/prompt-state.json` 不现实 · webapp 没法访问本地文件)。

实际接通到服务器存 muted/cooldown/dismissedToday 是 P1 (新加 `userPreferences` 表) · 这次先放占位说明: "通知偏好同步还在做 · 目前只在你装了 CLI 的机器上生效"。

#### 5.5 退出登录

按钮 · 调 `apiLogout` · 跳回 feed。

#### 5.6 删除账号

放最底 · vermilion 文案 · 二次确认 modal · 输入 handle 确认后才动。后端 deleteAccount API 本次 spec **不实现** (留 P1) · 前端按钮先存在 · 点击告知"还没接通 · 想删账号的话发邮件给我"。
