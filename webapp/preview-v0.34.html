关 于

捣鼓 / Tinker

给"用 AI 创造但不必懂代码"的人的工作室

GitHub 是给写代码的人的工作室。

我们是给捣鼓东西的人的工作室 — ta 们用 AI · 做出真的能跑的东西 · 但不必是程序员。

我们不做什么
  • · 不做推荐算法 / 热门 / trending
  • · 不做点赞总数 / 粉丝数 / 浏览量 这些"数字游戏"
  • · 不做等级 / 守护者 / verified 这些"身份证书"
  • · 不做 "X 天月入 $Y" 焦虑营销
我们做什么
— 给"在做但没做完"的人留主舞台

卡住、试错、放弃,都是 first-class 内容。不是只有"成功"才能展示。

— 真实复用反馈链

不是点赞,是「我用了你的方法 · 跑通了」。有人真的把你的东西接走用了,比 1000 个赞重要。

— 作品必须挂可访问产物

截图 + AI 文案 ≠ 作品。陌生人不联系你就能玩到的,才叫作品。这是反「AI 装大佬」的硬边界。

关 于 产 品 本 身

Tinker 用 Claude Code + Node.js + Docker 做的。

我们的开发过程本身就公开在这个产品里 — @daodao 工作室的「Tinker / 捣鼓」项目能看到每一次 commit、每一次卡住、每一次跑通。

这是产品的第一篇内容 · 也是它的第一个广告。
