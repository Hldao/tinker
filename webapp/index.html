关于

捣鼓·Tinker

给那种用 AI 做出能跑的小东西的人，不必懂代码。

GitHub 是给写代码的人用的工作室。

这里是给用 AI 做小东西的人用的工作室。他们做的东西能跑、能玩、能启发别人也动手做，不必懂代码，也不必证明自己"成了程序员"。

不做什么
  • 不做推荐算法，不做热门，不做 trending。
  • 不做点赞数、粉丝数、浏览量。
  • 不做等级、认证、verified 这种标记。
  • 不做"X 天月入 $Y"那种焦虑营销。
做什么

把"在做但没做完"的人当主角

卡住、试错、放弃，都跟"做完了"一样值得看。不是只有成功的事才能露面。

让别人真用上你的东西

不是点赞，是「我用了你的方法，跑通了」「看了你的进展，我也动手做了」。有人因为你的进展也动了手，比一千个赞重要。

作品必须真能玩到

不能只贴截图加 AI 写的介绍。陌生人不联系你就能玩到的，才叫作品。

做完的事也要被看见

改成「跑通了」时要写一句感想，进陈列馆。但仍然不投票、不排行，看到喜欢的就真的去试用。

这个产品自己

Tinker 是用 Claude Code 做的，跑在 Node 跟 Docker 上。

开发过程本身就公开在 @daodao 的「Tinker · 捣鼓」项目里，每一次卡住、每一次跑通都能看见。这就是它自己的第一份样品。
