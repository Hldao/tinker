## 五 · 还没改但**强烈建议**的 (超出 CSS 范围)

### 1. server seed 启动时自动 populate (中等优先级)
现在 `/tmp/tinker-local.db` 我手动 populate 了 starters + tools。但 **生产环境也可能漏掉**（如果你新建 DB 没走 `migrate-from-json.js`）。

**建议**: 在 `server/index.js` 启动时调一个 `ensureSeedTables()` 函数 · 检查 starters / available_tools 表为空就 populate（用 `seed.js` 里的 `STARTERS` / `AVAILABLE_TOOLS`）。

### 2. ESC 关 modal 列表补全 (低优先级)
现在 JS 第 3539 行 ESC 列表里没有 `modal-login` / `modal-wechat-qr` / `modal-set-handle`。
- `modal-set-handle` 不在是 by-design（强制流程）
- `modal-login` / `modal-wechat-qr` 应该加上

**建议** 改 JS line 3539 list 加这两个 id。v0.31 用 CSS ::after 加了 "esc" 暗示但实际 JS 不监听，加了 hint 反而 misleading。我可以补一个 JS 修改但用户原则是 CSS-only，记下来等用户决定。

### 3. shareText em dash 替换 (微小)
JS line 3470 分享文案 `@${owner} on Tinker / 捣鼓` 用 em dash。建议改 `·` 跟整站节奏一致。

### 4. 死代码清理 (清理性质)
`@keyframes anchorFlash` (原 webapp CSS line 370-374) 已不被任何 `.flash-anchor` 使用。落地时可删。

### 5. mention 正则中英不一致 (产品 bug，不在视觉)
- `renderMentions` (line 2127) 只识别 `@[A-Za-z0-9_]+`（不含中文）
- server `extractMentions` (`actions-sql.js` line 22) 识别 `[A-Za-z0-9_一-龥]+`（含中文）
- 后果：用户输入 `@张润江` 时 server 会发 `mentioned` 通知，但 webapp 渲染时 `@张润江` 不会变成可点 mention（光秃秃文字）。
- 不是 CSS 能解决的 · 但是有点尴尬
